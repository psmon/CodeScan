CodeScan 은 CLI와 TUI 두가지 인터페이스를제공
- CLI : Command Line Interface
- TUI : Text User Interface

추가 기능으로
project-update 에 --path 업데이트만 있는데  --source 업데이트도 추가지원

소스업데이트 규칙
- FullScan을 해서 전체 업데이트
- 업데이트 수행전 업데이트경로 프로젝트 git이용 가능한지 확인후 대상 프로젝트 pull해서 최신으로
- git을 이용못하거나 pull에 권한오류 또는 충돌등이 발생하면(예외) 경고오류-최신소스로 업데이트해주세요 남기고 진행한다. (옵셔널임)

# 세부지침
- CLI기능부터 만든후... TUI는 CLI가 작성한 코어모듈을 이용하게한다. (동일기능 중복구현 방지)
- 추가된 기능으로 help 업데이트를 필수로한다.
- TUI의 위치는 기존 TUI를 파악해 적절한 메뉴에 위치시킨다.

# 테스트
-영향받는 범위의 클래스를 수정했다고하면 추가된기능을 포함 관련 범위 유닛테스트 작성
  - 해당 클래스에대한 유닛테스트가 없는경우 신규작성(추가된 기능만 작성하지말고 함께포함된 함수도 테스트추가-없는경우)
  - CLI를 통해 등록,제거,스캔등 변경범위의 영향받는 부분을 테스트 
-유닛테스트가 통과되고나면 다음테스트
-테스트 대상 : CLI , TUI 모두테스트 (TUI는 화면캡쳐가 아닌 콘솔텍스트 캡처해 지원되는 키를 전송하면서 테스트 E2E)
 - 이 프로젝트의 절대경로를 파악해 추가한기능이 잘작동하나 테스트수행
   - CLI를 통해 등록,제거,스캔등 변경범위의 영향받는 부분을 테스트
