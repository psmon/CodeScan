# v1.6.0 — web-gui-design-reviewer 합류 + GUI Design Review 엔진

## 변경 요약

사용자 명시 요청 — "CodeScan의 web GUI 뷰(`Home/harness-view/`)의 디자인과
인터랙티브 개선, 디자인 지원은 `C:\code\psmon\pencil-creator` 디자인 하네스에서
지원" — 을 Mode B (Suggestion Tip)로 받아 새 에이전트 **web-gui-design-reviewer**
+ 새 지식 **web-gui-design-craft** + 새 엔진 **gui-design-review** 를 함께 합류.

정원이 9명의 전문가로 확장 (v1.5.0 8명 → v1.6.0 9명).
기존 8명은 전부 백엔드/파서/그래프 계열이었고, **웹 GUI 디자인·인터랙션**을
보는 첫 번째 꽃이다.

## 추가된 요소

### 에이전트 (agents/)

| 이름 | 페르소나 | 책임 |
|------|---------|------|
| `web-gui-design-reviewer` | 정원의 안목 (Aesthete) | `Home/harness-view/` 뷰어의 디자인 완성도·인터랙션 품질을 pencil-creator의 Case W 3축으로 검수. Playwright 캡처 + Pencil MCP .pen 목업으로 개선안을 시각적으로 제안하되, 실제 코드 수정은 harness-view-build CREATOR에 위임(역할 분리). |

### 지식 (knowledge/)

- `web-gui-design-craft.md` — pencil-creator의 `design-craft.md`(Case W) 평가
  철학을 정적 SPA 뷰어 맥락으로 번역. Case W 3축(W1 디자인 요소 커버리지 /
  W2 인터랙션 충실도 / W3 창의적 확장), 디자인 토큰 체계, 마이크로 인터랙션
  패턴, a11y(WCAG AA/포커스/`prefers-reduced-motion`) 체크리스트, Pencil MCP
  목업 워크플로우, 위임 경계를 단일 출처로 정의.

### 엔진 (engine/)

- `gui-design-review.md` — 캡처 → 3축 평가 → 개선안+목업 → 위임 적용 → 재검증
  순의 오케스트레이션. 입력 범위(전체/변경분) 분기. 참여: web-gui-design-reviewer
  (주축), 협력 스킬 harness-view-build(DEV SERVER/CREATOR) · playwright-e2e · Pencil MCP.

## 디자인 하네스 브릿지 (pencil-creator)

| 원천 (`C:\code\psmon\pencil-creator`) | 활용 |
|------|------|
| `.../knowledge/design-craft.md` | Case W 3축 평가 기준의 모태 |
| `.../agents/design-evaluator.md` | 깊은 디자인 평가 시 참조 |

Pencil MCP는 이 세션에 이미 연결되어 있어 목업 생성이 즉시 가능.

## 역할 경계 (오염 방지)

- **코드 수정 없음**: 이 에이전트는 CSS/JS를 직접 고치지 않는다. 무엇을·왜
  고칠지 판단하고, 파일·치환 명세를 `harness-view-build` CREATOR에 위임한다.
- 기존 `harness-view-build` 스킬(BUILDER/CREATOR/DEV SERVER)과 책임이 겹치지
  않도록, 이 꽃은 **안목(판단)**, 그 스킬은 **손(실행)**으로 분리.
